Introduction

What is Robotick?

The Robotick project was developed primarily as an initiative within Turkcell. It is an automation orchestration tool that automates daily tasks for the needs of operations teams.

Robotick does not require installation and can be accessed through a single web browser. You can automate your repetitive or periodic tasks using Robotick.

This documentation is to help you regarding each step of your Robotick experience. Please go through the documentation and watch the tutorial to understand the building blocks of Robotick and how to make most of its application.

Requirements

You will need the following to be able to use Robotick

  1. Connection to Turkcell network (direct connection or through VPN)
  2. Web browser
  3. Access to Robotick application

To start

Type robotick\ in the URL section of your browser, and that will take you to login page of the application

Use your LDAP username and password to log in.

To be able to operate on Robotick, you must be involved in a project. If there is no Project Manager User who can include you in the Project, you can send an e-mail to Robotick team to be included in a project. (TEAM-RPA-CORE <TEAM-RPA-CORE@turkcell.entp.tgc>)

If you are not involved in any project, an empty interface will welcome you after login.

Terminology #back to top

Project #back to top

    Project refers to the project the user is working in. One user can have one or more projects to work on. Each project can have multiple flows.

    Flow #back to top

      Flow refers to the workflow. Flow contains components and the connections between them, indicated with arrows. The flow works in the direction of arrows, starting from the Start component, and ending at the End component in the flow editor. The user can create flows in the Flow Editor by putting components together and making connections between them.

    • A Flow has the following properties:
      • ID -The unique ID number of the Flow
      • Flowname - name of the flow
      • Status - the status of the flow. For example: "deployed"
      • Schedule - the schedule of the flow.

      • Flow List #back to top

        The user can view the information of their flows in the Flow List.

        Flow Editor #back to top

          Flow editor is where the user can create a flow and edit or trigger it. To start creating a flow, the user can select a component, drag and drop to the editor. The user can then connect different components by clicking the green dots in a component, which produces an arrow, and connect it to another component. In the flow editor, the user can create one flow which has to start with the Start component and end with the End component. The direction of the flow is determined by these two component: the flow starts with the Start component and ends with the End component. Every other component comes in between these two components and need to be connected to each other (with no dangling component in the editor). As the user establishes connections between components, he/she determines the direction of the flow as well. The arrows that make the connections between components indicate the direction of the flow.

          Components #back to top

            Components are the building blocks of a flow. They are the objects that make up the flow. Every component object has different functionality. The components have parameters to tune that component for the exact purpose the user wants. The user can set the parameters by clicking the component in the flow editor and filling the parameters that show up on the right hand side of the page. Some (not all) components have return values as well. The user can see the return values by clicking the component in the flow editor and scrolling down the page.

            Components #back to top

            DATABASE #back to top

            The following components work with database operations

            1. Select

              The SELECT component is used to make SELECT queries on a database connection. It needs a DB Connection component beforehand.

                Parameters:
              • COMPONENT NAME - name of the component. It is generated automatically.
              • CONNECTION - Choose from existing DB connections
                • For example:
                  • MY_NAME
              • SQLSTRING - SELECT SQL statement to be run
                • For example:
                  • select to_char(sysdate, 'dd.mm.yyyy HH24:MI:SS') as TIME from MY_ADMIN.CHECK_DATA group by substr(user_nname,1,7)
                Returns:
              • RESULTSET variable - type TResultSet
              • RESULTCOUNT variable - type NUMBER
            2. PLSQLBlock

              The PLSQLBlock component is used to run PLSQL commands on a database connection. It needs a DB Connection component beforehand.

                Parameters:
              • COMPONENT NAME - name of the component. It is generated automatically.
              • CONNECTION - Choose from existing DB connections
                • For example:
                  • DBConnection_1_CONNECTION
              • PLSQLSTRING - SQL Statement to be executed in the Connected DB.
                • For example:
                  • BEGIN BILLING.RPA_UTILS_PKG.EXEC_Q4_GET_ALL_RECORDS(4); END;
                Returns:
              • PLSQLRESULT variable - type STRING
            3. DBConnection

                It establishes a connection to databases. Once you have established the connection, you can use the same connection as many times as you want. You can run SQL query.

                Parameters:
              • COMPONENT NAME -name of the component. It is generated automatically.
              • DB TYPE -database type
                • For example:
                  • Oracle Connection
              • HOSTNAME - IP addres to connect
                • For example:
                  • 123.456.78.90
              • PORT -database port
                • For example:
                  • 1234
              • SERVIS ADI - SERVICENAME of the connected DB
                • For example:
                  • abcdef
              • USERNAME -Username for DB connection
                • For example:
                  • TCUSERNAME
              • PASSWORD - Password for DB connection
                • For example:
                  • my_password
                Returns:
              • CONNECTION variable - type TDbConnection
            4. ExportResultSet

              The ExportResultSet component is used to Export TResultSet query results from SQL queries into Excel or Email.

                Parameters:
              • COMPONENT NAME -name of the component. It is generated automatically.
              • RESULTSET - The SQL Select components that create the TResultSet output in Flow will be listed here. Select the query result you want to export.
                • For example:
                  • SELECT_1_RESULTSET
              • OUTPUT TYPE - Select the format in which you want to export the query result. Options: Excel, HTML.
                • For example:
                  • HTML
                • Note: HTML option will generate output in THTML format that allows you to embed the query result in the email body part as a table. You can use this output by adding $ {ExportResultset_xXx_HTML} to the body part of the send email component. You can use it by adding the THTML output name. You can see the THTML output name in component Flow variables.
                Returns:
              • EXCEL variable - type TFILE
              • HTML variable - type THTML

            EMAIL #back to top

            The following components work with email operations

            1. ConnectMailbox

              The ConnectMailbox component performs the email server connection required for Robotick to read your emails.

                Parameters:
              • COMPONENT NAME -name of the component. It is generated automatically.
              • URL -URL of the mailbox
                • For example:
                  • name.surname@turkcell.com.tr
              • USERNAME -username to log in
                • For example:
                  • TCUSERNAME
              • PASSWORD -password to log in
                • For example:
                  • my_password
                Returns:
              • SESSION variable - type TSession
            2. SendEmail

              The SendEmail component allows you to send Email. Dynamic parameters and file attachment can be made to sent emails. Body and subject sections can be sent parametric emails with the format $ {Another Component Output}. An email can be sent to more than one user by adding semicolons between the addresses in the To parameter.

                Parameters:
              • COMPONENT NAME -name of the component. It is generated automatically.
              • TO - e-mail adress which will receive email
                • For example:
                  • TEAM-ADMIN@turkcell.com.tr
              • SUBJECT - subject line
                • For example:
                  • LAST WEEK REPORT
              • BODY -body of the email
                • For example:
                  • Merhabalar, Son dosya kontrolu buradadir: ${ExportResultset_1_HTML}
              • FILES - Files in this flow you can attach to the e-mail
                • For example:
                  • ExportResultset_1_EXCEL
                Returns:
              • returns nothing
            3. ChangeMailStatus

              The ChangeMailStatus component can change the status of the email in the folder you specified in the ReadMail_1_ID parameter, which is the output of the ReadMail Component, to read / unread and move it to the folder you specify.

                Parameters:
              • COMPONENT NAME -name of the component. It is generated automatically.
              • SESSION - Email box link you want to read. It comes as the output of the Connect Mailbox component.
                • For example:
                  • ConnectMailbox_1_SESSION
              • ID - The email ID whose status you want to change or move. It comes as the output of the ReadMail component.
                • For example:
                  • ReadMail_1_ID
              • CHANGE READ STATUS - What do you want to do with the status of the email? READ / NOT READ.
                • For example:
                  • READ
              • MOVE FOLDER - Do you want to move the email to another folder? If it is ON, yes.
              • DESTINATION FOLDER - The folder to which the email is to be moved
                • For example:
                  • OTOMASYON_FOLDER
                Returns:
              • RESULT variable - type STRING
            4. ReadMail

              The ReadMail component allows Robotick to find and read the mails in the folder you specified in your mailbox according to the words you specify.

                Parameters:
              • COMPONENT NAME -name of the component. It is generated automatically.
              • SESSION - Email box link you want to read
                • For example:
                  • ConnectMailbox_1_SESSION
              • FOLDER - Email folder you want to check
                • For example:
                  • ONEDESK
              • FROM - The sender email address you want to check
                • For example:
                  • username@turkcell.com.tr
              • SUBJECT CONTAINS - Text in the subject section
                • For example:
                  • operasyon
              • FROM DATE - Checks the mails received after the given date
                • For example:
                  • Select a date from the pop-up calendar.
              • BODY CONTAINS - Text fragment in body
                • For example:
                  • Action expected
              • TYPE OF MAIL - Checks for READ, UNREAD, or ALL mails
                • For example:
                  • ALL
                Returns:
              • ID variable - type STRING
              • MAILSUBJECT variable - type STRING
              • MAILBODY variable - type STRING
              • RESULT variable - type STRING
              • SENDER variable - type STRING

            DECISION #back to top

            The following componen work with decision operations

            1. IfElse

              The IfElse components is used to set up conditions for specific operations to be exefcutd, or not executed

                The IfElse component allows the Flow to follow different paths according to a specific parameter. Unlike other components, its parameter is defined not on the component, but on the connection between the component and the other component it is connected to. If else, if the condition in one direction is not met, it automatically enters the other empty branch.

                Parameters:
              • COMPONENT NAME - name of the component. It is generated automatically.
              • Condition - The blank input box where the user can type the condition
                • The following operators can be used:
                  • < less than
                  • > greater than
                  • == equals to
                  • != not equals to
                • The user can control different parameters from various components. For example:
                  1. FindString component
                      Find_result: 'true ', 'false' depending on the result found.
                    • For example: FindString_1_FIND_RESULT == true
                      Count: Number of results found
                    • For example: FindString_1_COUNT == ‘4’
                      Find_Out: Returns the entire line where the word is a match
                    • For example: FindString_1_FIND_OUT == ‘Bu satırda alarm kelimesi geçti’
                  2. LinuxCMD component
                      SSH_out: The text the command returned as a result
                    • For example: LinuxCMD_1_SSH_OUT == true ’, false
                      Retcode: Returns '1' if the command runs successfully, and '0' if unsuccessful.
                    • For example: Örnek: LinuxCMD_1_RETCODE == ‘1’
                  3. ReadMail component
                      Result: 'True' if the mail was found under the search conditions, false if not.
                    • For example: Örnek: ReadMail_1_RESULT == ‘ true
                Returns:
              • returns nothing

            STRING #back to top

            The following components work with string operations

            1. FindString

              FindString components allow to search text fragments within the output of components that make a String output. Thus, it can be checked whether certain keywords are in the relevant text.

                Parameters:
              • COMPONENT NAME - name of the component. It is generated automatically.
              • SOURCE STRING - The output values provided by the components that output String are automatically added here. Select the output you want to search for text.
                • For example:
                  • GetDate_1_OUTPUTDATE_STR
              • FIND STRING - The text to be searched is written here in single quotes. RegulerExpression can be entered here without quotes.
                • For example:
                  • 'Error:404'
                • RegularExpression can be entered here without quotes. For example:
                  • .*^(10|20|25).*
                  • (? <=s ) d+(.*)(?=nolu.*güncel fatura)
              • REGULAR EXPERESSION- If ON, the string user wants to find (FIND STRING input) will be considerd as a regular expression input
              • EXACT MATCH - If ON, the string user wants to find needs to be exact match with the found string
              • CASE SENSITIVE- IF ON, the string user wants to find will be case sensitive
                Returns:
              • FIND_RESULT variable - type STRING
              • FIND_OUT variable - type STRING
              • COUNT variable - type STRING

            OS #back to top

            The following components work with OS operations

            1. LinuxCMD

              LinuxCMD components enable the ability to run commands by establishing connections on Linux servers.

                Parameters:
              • COMPONENT NAME - name of the component. It is generated automatically.
              • HOSTNAME - IP Address of the connected Server
                • For example:
                  • 12.345.678.99
              • COMMAND - Linux Shell command
                • For example:
                  • dzdo -iu intconn /myapp/folder/BIN/file.sh ABCD_IN
              • USERNAME - Username in the connected server
                • For example:
                  • tc000000
              • PASSWORD - Password for the username
                • For example:
                  • my_password_1123
                Returns:
              • SSH_OUT variable - type STRING
              • LinuxCMD_1_RETCODE variable - type STRING
              • LinuxCMD_1_FILES variable - type TFILEs

            DateTime #back to top

            The following components work with date and time operations

            1. GetDate

              With GetDate components, you can convert the current date or a date you choose to a format you want to use in different server, operating system and database systems and use them parametrically.

                Parameters:
              • COMPONENT NAME - name of the component. It is generated automatically.
              • INPUT OPTION - With it System option, the GetDate component automatically receives the date and time when flow runs. With the User Defined oprion, it takes the date and time you selected in the Input Date section and always uses this date.
                • For example:
                  • SYSTEM
              • INPUT DATE - Allows you to manually select the date you want.
              • OUTPUT FORMAT OPTION - With its Prebuilt option, it allows you to choose one of the 12 different date and time formats available. With the Custom option, you can create your own date and time format with the rules in the tooltip below.
                • For example:
                  • CUSTOM
              • OUTPUT FORMAT - Output date and time format
                • For example:
                  • dd
                Returns:
              • OUTPUTDATE variable - type DATE
              • OUTPUTDATE_STR variable- type STRING

            Triggering the flow #back to top

              The user can trigger the flow in the Flow Editor by clicking the buttons below:

              DEPLOY #back to top

              This button sends the flow to the backend, ready to be run. The flow can be run only after it is deployed.

              RUN #back to top

              This button runs the flow. User can click this button only after delpoying the flow using DEPLOY button.

              SAVE #back to top

              Saves the flow

              CANCEL #back to top

              This button quits the Flow editor.

              Types of Users #back to top

              Here are the types of users and their access rights:

              1. Root Admin
                • Has access to: anything and everything
              2. Project Admin
                • Has access to: Project list, flow editor, flow list, execution history, tutorial, create new project, user managemenet

                • Has access to: Project list, flow editor, flow list, execution history, tutorial
              4. Monitor User
                • Has access to: Project list, flow list, execution history, tutorial

              Add a new user #back to top

              When you click Settings on the main menu, User Management page comes up. To add a new user, the user needs to type the Username and Password, select the project and the role. When the user clicks on "Add a new user" , the user is added to the project.

              Edit an existing user #back to top

              In User Management page, users of the project are displayed in a table. If the user has authorization, user can edit or delete the users using Edit or Delete buttons near the user information in the table.
